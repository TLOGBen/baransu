什麼是 sequence diagram

Sequence diagram 不是流程圖。它強調的是時間順序 + 跨 actor 互動：時間軸從上往下，每條垂直 lifeline 代表一個 actor 或 participant，水平 arrow 是訊息。同步呼叫用實線實心箭頭；回傳用虛線。Actor 在處理工作時，lifeline 上會疊一條細長的 activation bar 標記「現在在 busy」。Actor 對自己呼叫的內部處理用 self-loop（一段右出 → 下 → 左回的折線）。

用得對的場景：3 個以上 actor、有 round-trip、互動順序本身就是要傳達的重點。例如 OAuth flow、API 編排、microservices 對話、CI/CD pipeline 內外部呼叫。不適合：純粹的決策樹（用 flowchart）、靜態系統結構（用 architecture）。

跟 flowchart 的分工

判讀方法：多 actor + 時序往下走 = sequence；單一系統 + 決策分支 = flowchart。

幾何完全不動的證據

三輪 preset 切換，所有座標 / viewBox / 節點寬 / message y / activation bar 位置完全等同。viewBox 720×384、4 actor at (48 / 216 / 384 / 552, 48, 128×44)、3 activation bar at /book(208h) / markitdown(32h) / validate-output(32h)、8 message y=124/156/188/216-224/244-252/272/300/328、focal cap=2、GATE A-L 全綠。

換 preset 不動結構，是 v1.3 canonical token schema 的勝利條件。如果同一份內容換 preset 後幾何要重算，那就是 preset coupling 還沒堵乾淨 — 三輪 dogfood 反向驗證 v1.3 把它堵掉了。

三 preset 哲學差

同一份 sequence diagram 在三 preset 之間切換，傳達的氛圍不同：

  • Kami — 紙墨溫度，serif 字身，ink-blue 收斂；像把 sequence diagram 印在羊皮紙上的筆記
  • Material 3 — 表達性、無障礙、surface tint elevation；像 Android Studio 內的 component preview
  • Swiss IKB — Müller-Brockmann 網格、單一強調色、heavy rules；像 1960s ETH Zurich technical poster

同樣的 4 actor 8 message，三個 preset 下分別感覺像「閱讀筆記 / 互動設計稿 / 技術海報」— 視覺哲學承載資訊以外的身分。
