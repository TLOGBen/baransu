book diagram port v1 — Design

book skill 把 Kami 圖表規格移植進來時的內部設計：四層架構承載 instruction / spec / template / validation / asset；token 採三層 fallback；Stage 2 用 content-type × 13-type 兩階層正交決策樹；驗證 gate 守住結構與引用一致性，圖檔遵守 paper-mask + chevron-only 的 WeasyPrint-safe 規格。

01 四層架構

book skill 移植後分成五個內部層，每一層各自有清楚的職責邊界。指令層只規範行為流程，不寫死視覺；規格層放設計約束與 type ref；模板層是 show-by-example 的 HTML + SVG defs 範本；驗證層守 gate；資產層放真實 example HTML。

02 整體流程

流程沿用 book skill 既有 Stage 0→4 骨架，v1 在中間插兩個新節點：Stage 0.5 token resolver（讀 root DESIGN.md / 回退 Kami preset）和 Stage 2A 的兩階層分類（內容類型 → 13 型 diagram selection → per-diagram lookup）。其餘 stage 不動。

資料流上，$RAW_CONTENT 經 content-type 分類器產出 A/B/C 標籤，同時 13 型 selection 為每段內含 diagram 的 section 派一個 diagram type。root DESIGN.md（可選）和 Kami preset 共同進 token resolver，per-type derived rule 補上 sequence / state / swimlane / er 四型的特殊需求。最後 render 出 SVG，驗證層守關，輸出 .claude/book/{slug}.html。

兩個新節點都是 v1 才加，未來若 perception-guide 需要回流，Stage 2A.1 的分類器是第一個被改的點。

03 Token 三層 fallback

Token 取得走嚴格的 fallback waterfall：Layer 1 看 root DESIGN.md 有沒有 token，且值要通過 hex regex；Layer 2 退到 Kami preset；Layer 3 對特定 type（sequence / state / swimlane / er）跑 derived rule。Layer 1 失敗（非 hex 或 injection-shape）就 silent fallback 並記 warning — 不暴露在最終輸出。

04 SVG primitives

四條 primitive 規範把 Kami 圖表的視覺簽名固定下來。每張 SVG 開頭必加 <defs> 內三個 marker（雖然 WeasyPrint marker 支援不可靠，所以 v1 實際輸出 chevron 用 <path>，<marker> 只作為 future-proof 容器）；節點背景採兩層 paper-mask；節點左上 7px Geist Mono uppercase 小 type tag；底部 ~60px legend strip 含 hairline rule + LEGEND text + items。

每張 SVG 開頭必加

<defs>
  <pattern id="dots" width="22" height="22" patternUnits="userSpaceOnUse">
    <circle cx="1" cy="1" r="0.9" fill="#E3E2DC"/>
  </pattern>
</defs>
<rect width="100%" height="100%" fill="#f5f4ed"/>
<rect width="100%" height="100%" fill="url(#dots)" opacity="0.55"/>

三個 marker：arrow（default muted）、arrow-accent（focal / 主流，brand 色）、arrow-link（external / API call，brand-light）。Type tag 範例："API" / "DB" / "EXT" / "CACHE"，0.8 stroke 細框。Legend strip 在 SVG 底部 hairline + LEGEND text + horizontal items（每項一個 mini swatch + label）。

06 錯誤處理與邊界

驗證層分三類 gate — 結構 / 引用一致性 / deny-list。三類任一 FAIL 都停輸出。Token resolver 的 hex regex 不只擋格式錯，也擋 injection-shaped 注入（red;}</style><script> 視為非 hex 同類處理）— security critical。

與 /design skill 的關係

v1 為單向讀取：/design 寫 root DESIGN.md，/book 讀。/book 不主動通知 /design「我在用哪些 token」，也不要求 /design 適配 book 的 13 型。v2-N 若需雙向，另起 issue。

單向依賴是邊界誠實的表現。雙向耦合會讓兩個 skill 的演進互相鎖死，v1 拒絕走這條路。
