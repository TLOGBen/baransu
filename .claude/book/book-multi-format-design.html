01三層架構：指令、腳本、模板

book skill 由三個職責分明的層次組成。指令層是 Claude 的執行劇本，腳本層是機器可呼叫的工具，模板層則作為「show-by-example」合約，Claude 在生成 HTML 時以模板對齊基準，而非自由發揮。

1指令層

SKILL.md — 定義 Stage 0–4 的行為規格，Claude 依循執行。

2腳本層

機器可執行的工具腳本；shell 呼叫，exit code 為閘門訊號。

3模板層

golden / slide template — show-by-example，禁止發明新 CSS pattern。

02整體操作流程

Pipeline 由 5 個 stage 串成。Stage 0 解析 --format 並安裝依賴；Stage 1 取得原始內容；Stage 2 拆成 2A（長文）與 2B（投影片，條件執行）；Stage 3 渲染並依旗標分支生成 PDF / PPTX；Stage 4 跑 validate-output.ts gate 並用 browser-use 截圖。

$SLUG 推導唯一性：$SLUG 只在 Stage 2A 推導一次，Stage 2B 與所有 Render 步驟繼承同一值，不另行推導。這保證三種格式輸出檔名一致。

03雙軌資料流：長文 vs 投影片

原始內容 $RAW_CONTENT 同時餵給兩條 Synthesize 軌道。長文軌產出 $STRUCTURE_LONGFORM（4–8 節，每節含 claims 與 SVG flag）；投影片軌產出 $STRUCTURE_SLIDES（6–12 張，layout_type 枚舉）。兩軌共用 $SLUG，但渲染目標完全獨立。

04SVG 精度規格（設計層約束）

SVG 是這份設計的「抗 slop」防線。所有圖形必須走解析度離散化的座標系：節點寬只能是 128 / 144 / 160，高度只能是 32（pill）或 64（standard），所有座標為 4 的倍數。顏色禁用 rgba() 於 fill / stroke，一律使用 solid hex token。

約束項目 規格
顏色表示Solid hex token；禁 rgba() 於 fill/stroke
座標系所有數值為 4 的倍數
節點寬128 / 144 / 160（三檔離散）
節點高32（pill）/ 64（standard）
焦點節點最多 1–2 個（--brand 描邊 + --brand-tint 填色）
箭頭chevron path；禁 <marker orient="auto">
text top padding<text y> ≥ font-size × 1.2
嵌入字體14–24（嵌入 A4 後 scale ≈ 0.47）
必備 defsparchment rect + dotted-noise pattern
為什麼是離散化？連續座標讓 LLM 自由發揮，產出對齊歪斜、間距失控；離散化把選擇壓縮到「三檔寬 × 兩檔高」，視覺一致性大幅提升，同時不犧牲表達力。

06錯誤處理策略

錯誤處理採「分層阻斷 + 部分失敗繼續」原則：Stage 0 級錯誤（依賴缺失、旗標不合法）直接停止；Stage 3 級錯誤（PDF / PPTX 生成失敗）只標記該格式失敗，其他格式繼續執行。

錯誤場景 處理層 行為
WeasyPrint 安裝失敗Stage 0錯誤 + 手動指令，停止
Playwright / pptxgenjs 安裝失敗Stage 0錯誤 + 手動指令，停止
--format 不合法Stage 0列出合法值，exit 1
slide HTML 驗證失敗Stage 3§6標記 PPTX 失敗，其他格式繼續
validate-output.ts GATE FAILStage 4修正後重跑，第二次失敗停止
html2pptx.js 執行錯誤Stage 3§6輸出錯誤，PPTX 標記失敗
$SLUG 同名衝突Stage 2A加 _v2 後綴，所有格式同步
WSL2 Playwright 需 --no-sandboxGotchas提示手動補旗標
設計取捨：「部分失敗繼續」讓使用者 --format all 時仍能拿到可用的格式子集，而不是因為一種格式失敗就整批作廢。HTML 永遠是最後一道保障 — 即便 PDF / PPTX 都失敗，HTML 仍會產出。
