從 /baransu:learn 的 Kami digest 與 huashu-design 研究出發，記錄 /baransu:book 從單一 HTML 路徑進化成 multi-format（HTML / PDF / PPT）pipeline 的決策軌跡，以及把 SVG 精度規格抽出 design / book 兩 skill 邊界的研究筆記。

01 看了什麼

這份 WIP 起點是四份 reference material：Kami digest（/baransu:learn 的產物）、book skill 目前的 SKILL.md（v1.0）、huashu-design 的 html2pptx.js 架構研究、以及 design skill 已落地的 紙-preset DESIGN.md。先盤點，再決定 book 端到底要動什麼、哪些可以不動。

/baransu:learn 的 Kami digest

最關鍵的發現是：Kami 的所謂「3D 圖」並非代碼渲染，而是 ChatGPT Images 2.0 依據 references/ 作為 design brief 生成的視覺。PDF 路徑是 HTML → WeasyPrint → PDF，沒有另起爐灶；但 WeasyPrint 帶兩個必避的 bug — rgba() 在 tag 背景觸發 double-rectangle 渲染，必須換 solid hex；<marker orient="auto"> 不支援，箭頭會全部朝右，必須改成手工 chevron <path>。SVG 圖表是自包含 inline，14 種類型走 first-match 決策樹，並有嚴格的「防 AI slop」精度約束。

book skill v1.0 SKILL.md（現況）

Stage 3 §4 的 SVG 規格過鬆：用了 rgba(27,54,93,0.07) 做節點背景違反 Kami 不變量 #8、沒有坐標 ÷4 規則、無節點寬三層白名單、無 focal 節點上限、用 <marker>、無嵌入字體校正、無 <defs> 共用片段。Stage 2 §3 也沒有任何圖表類型路由，只說「decide if beneficial」。輸出格式只剩 HTML，截圖是驗證手段而非正式產物。

huashu-design 的 html2pptx.js

huashu-design 的 PPTX pipeline 是 HTML → Playwright（Chromium headless）→ pptxgenjs → .pptx。html2pptx.js 讀 DOM computed styles，把每個元素對應成真正的 PPTX text frame 物件，不是截圖 image-bed。canvas 尺寸 LAYOUT_WIDE 13.333"×7.5"（960×540pt）。相依只有 playwright + pptxgenjs，跨平台、無 LibreOffice 依賴。這是 Route D，優於 python-pptx（排版繁瑣）與 LibreOffice（環境依賴 + 排版飄移）。結論：book skill 的 PPT 路徑採 Route D。

design skill + 紙-preset DESIGN.md

design skill 已完整涵蓋前半段風格：Kami 十不變量、完整色彩 token、字型堆疊、元件樣式、間距系統、SVG diagram 色序。結論：book 不需要重造色彩系統，只需 reference design 的 tokens、補上 SVG 圖表的技術精度規格。

04 SVG 精度規格

這是 book skill 必須自己負責的部分。color token map 來自 design skill 的紙-preset，套到 diagram 的角色語義上：

SVG 角色token值
Canvas--paper#f5f4ed
標準節點填色--surface#faf9f5
標準節點描邊--text-primary#141413
焦點節點填色--brand-tint#EEF2F7
焦點節點描邊--accent#1B365D
標準箭頭--text-muted#504e49
焦點箭頭--accent#1B365D

抗 slop 約束（hard）

所有坐標、寬度、間距 4 的倍數。節點寬只有三層 128 / 144 / 160（小圖可壓兩層）。節點高 32（pill）或 64（standard）。焦點節點上限 1-2 個。禁用 <marker orient="auto">，改手工 chevron。箭頭 endpoint 精確落在節點邊緣，無 gap。SVG top padding：<text y="..."> 的 y ≥ font-size × 1.2 防字母頂部截斷。

共用 <defs> 片段（每張 SVG 開頭必加）

<defs>
  <pattern id="dots" width="22" height="22" patternUnits="userSpaceOnUse">
    <circle cx="1" cy="1" r="0.9" fill="#E3E2DC"/>
  </pattern>
</defs>
<rect width="100%" height="100%" fill="#f5f4ed"/>
<rect width="100%" height="100%" fill="url(#dots)" opacity="0.55"/>

嵌入字體校正：A4 長文嵌入後 scale ≈ 0.47，需放大。H2 / 焦點節點 24、Body / 節點 22-24、H3 / 子標籤 18-20、Caption 15-16、Mono uppercase tag 14。

05 PDF · PPT pipeline

PDF 直接走 WeasyPrint — 條件是 HTML 已經 WeasyPrint-safe（無 rgba tag 背景、無 <marker orient="auto">），上一節的 SVG 規格已經處理。PPT 走 Route D。下面是 Route D 的 pipeline：

Slide HTML 約束（4 條 hard）

配合 pptxgenjs 的 PPTX 物件對應，slide HTML 有四條 hard constraint：(1) body width = 960px、height = 540px；(2) 文字只用 <p> / <h1>–<h6>，不直接放在 <div>；(3) 禁用 CSS gradient（改 solid hex）；(4) 禁用 background-image（改 <img> 標籤）。
