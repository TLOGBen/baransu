# Render Pipelines — Stage 3 §5 (PDF) & §6 (PPTX)

Read this file only when `--format` includes `pdf`, `ppt`, or `all`.

## §5 PDF pipeline (`--format` includes `pdf` or `all`)

Run only when `$FORMAT` is `pdf` or `all`.

**Step 1: HTML preprocessing**

Take the long-form HTML content generated by Stage 3 §2-§4 and inject the following `<style>` at the end of `<head>`:

```html
<style>
  .toc-wrap { display: none; }
  @page { margin: 2cm; }
  body { font-family: var(--font-serif); }
</style>
```

> v1.3 uses canonical `var(--font-serif)` instead of a hardcoded font stack; the variable is resolved from the project root tokens.css (kami → serif stack; swiss/google-design → sans alias).

Save the modified HTML to a temporary path `{patched_html}` (e.g. `.claude/book/{$SLUG}-pdf-patch.html`).

**Step 2: Call WeasyPrint**

```bash
python3 -m weasyprint "{patched_html}" ".claude/book/{$SLUG}.pdf"
```

If the command fails (exit code ≠ 0): output the warning `⚠️ PDF 生成失敗：WeasyPrint 錯誤`, continue with the other formats, and do not stop the whole flow.

## §6 PPTX pipeline (`--format` includes `ppt` or `all`)

Run only when `$FORMAT` is `ppt` or `all`. Depends on the `$STRUCTURE_SLIDES` generated by Stage 2B.

**Step 1: Generate slide HTML**

For each slide object in `$STRUCTURE_SLIDES`, read the skeleton from `{project_root}/slide-cores/<layout-id>.html` (`<layout-id>` = the Stage 2B dynamic decision-table result, e.g. `cover.html` / `content-bullets.html` / `closing.html`) and generate a slide HTML file.

If `{project_root}/slide-cores/<layout-id>.html` is missing (consistent with the Stage 2B graceful-degradation behavior): warning「請先跑 `/baransu:design preset <style>` 取得 slide-cores」, and the body slot degrades to the inline skeleton of the hardcoded fallback three layouts (`cover` / `closing` / `content-bullets`); do not abort Stage 3.

Output spec for each slide:

- `<body style="width:960px; height:540px; margin:0; padding:0;">`
- Each slide is wrapped in `<div class="slide" data-layout="{layout_type}">`
- Text content is rendered with `<h1>`/`<h2>` and `<ul><li>`
- If `has_svg` is true, insert the corresponding inline SVG
- **Large type size and minimum font size**: apply the three hard rules from `slide-synthesis.md` §「投影片字級與限高硬規則」 (large-type dual constraint `min(Xvw,Yvh)` with `Y≥X×1.6`, the Chinese-title tier table, and the 18/16/14px minimum font-size floors); violating any one is marked fail in the final-report self-check — fix the copy or split the page before rendering.

**Step 2: Validate the slide HTML**

Before calling html2pptx.js, validate three things:

1. The `width` style of `<body>` contains `960`
2. The document contains at least one `.slide` element (`class="slide"`)
3. It does not contain `background-image`

If any check fails: output `⚠️ Slide HTML 驗證失敗：{失敗原因}`, do not call html2pptx.js, and continue with the other formats.

**Step 3: Call html2pptx.js**

```bash
node "$CLAUDE_SKILL_DIR/scripts/html2pptx.js" "{slide_html_path}" ".claude/book/{$SLUG}.pptx"
```

If the command fails (exit code ≠ 0): mark `PPT：失敗（詳見上方錯誤）`, continue with the other formats, and do not stop the whole flow.
