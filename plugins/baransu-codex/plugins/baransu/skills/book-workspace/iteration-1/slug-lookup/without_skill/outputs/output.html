Context Engineering

The art and science of filling the agent context window with just the right information at each step.

TL;DR

Agents need context to perform tasks. Context engineering is the art and science of filling the context window with just the right information at each step of an agent's trajectory. This post breaks down four common strategies — Write, Select, Compress, and Isolate — drawing on popular agents and research papers, then shows how LangGraph is designed to support each one.

What Is Context Engineering?

As Andrej Karpathy puts it, LLMs are like a new kind of operating system. The LLM is the CPU, and its context window is the RAM — the model's working memory. Just like RAM, the context window has limited capacity, and just as an OS curates what fits into RAM, context engineering curates what fits into the context window.

"…the delicate art and science of filling the context window with just the right information for the next step." — Andrej Karpathy

Three Types of Context

  • Instructions — prompts, memories, few-shot examples, tool descriptions
  • Knowledge — facts, memories, retrieved documents
  • Tools — feedback returned from tool calls

Context Engineering for Agents

Agents interleave LLM calls and tool calls, often for long-running tasks. Accumulating tool feedback means agents can consume a large number of tokens, leading to several problems:

  • Exceeding the context window capacity
  • Inflated cost and latency
  • Degraded agent performance

Four Ways Context Goes Wrong

Context Poisoning

A hallucination enters the context and propagates forward.

Context Distraction

Excess context overwhelms the model's training signal.

Context Confusion

Superfluous context steers the model off-course.

Context Clash

Contradictory parts of context pull in opposite directions.

"Context engineering is effectively the #1 job of engineers building AI agents." — Cognition
"Agents often engage in conversations spanning hundreds of turns, requiring careful context management strategies." — Anthropic

Four Core Strategies

✍️

Write

Save context outside the window for later use.

🔍

Select

Pull only the relevant context into the window.

📦

Compress

Retain only the tokens required for the task.

🌐

Isolate

Split context across agents or environments.

1. Write Context

Writing context means saving it outside the context window so the agent can access it later — within the same session or across many sessions.

Scratchpads

Like a human taking notes, agents can use a "scratchpad" to persist information outside the context window. This can be implemented as a tool call that writes to a file, or as a field in a runtime state object.

"The LeadResearcher begins by thinking through the approach and saving its plan to Memory to persist the context, since if the context window exceeds 200,000 tokens it will be truncated." — Anthropic, Multi-Agent Researcher

Long-Term Memories

Scratchpads help within a session; memories help across sessions. Reflexion introduced self-generated memories after each agent turn. Generative Agents synthesized memories from collections of past feedback. Products like ChatGPT, Cursor, and Windsurf all ship auto-generated long-term memories.

2. Select Context

Selecting context means pulling it into the context window when and only when the agent needs it.

Memory Types

  • Episodic — few-shot examples of desired behavior
  • Procedural — instructions that steer behavior
  • Semantic — facts for task-relevant context

Simple agents use a fixed set of files always in context (e.g., CLAUDE.md, Cursor's rules files). Larger memory stores require embeddings or knowledge graphs for retrieval, which introduces selection errors — as Simon Willison observed when ChatGPT unexpectedly injected his location from memory into an image request.

Tool Selection via RAG

When an agent has too many tools, overlapping descriptions cause confusion. Applying RAG to tool descriptions fetches only the most relevant tools per task — recent papers report up to a 3× improvement in tool selection accuracy.

Knowledge Retrieval (RAG)

Code agents illustrate large-scale RAG challenges well. Windsurf's Varun notes:

"Indexing code ≠ context retrieval … embedding search becomes unreliable as the codebase grows … we must rely on a combination of techniques like grep/file search, knowledge graph based retrieval, and a re-ranking step." — Varun, Windsurf

3. Compress Context

Compressing context involves retaining only the tokens required to perform a task.

Summarization

Summarization is the most common compression technique. Claude Code's "auto-compact" fires at 95% context utilization and summarizes the full trajectory. Strategies include:

  • Recursive summarization — progressively condenses older turns
  • Hierarchical summarization — structures summaries by scope
  • Boundary summarization — applied at agent-agent handoff points
  • Tool-call summarization — post-processes token-heavy tool outputs

Context Trimming

Whereas summarization uses an LLM to distill meaning, trimming uses hard-coded heuristics — for example, removing older messages from a sliding window. Research tools like Provence use a trained context pruner for question-answering tasks.

4. Isolate Context

Isolating context involves splitting it up so that each agent or environment handles only the portion it needs.

Multi-Agent Architectures

Splitting work across sub-agents gives each agent a focused context window. Anthropic's multi-agent researcher found that many specialized agents with isolated contexts outperformed a single agent — though at the cost of up to 15× more tokens.

"[Subagents operate] in parallel with their own context windows, exploring different aspects of the question simultaneously." — Anthropic

Sandboxed Environments

HuggingFace's deep researcher uses a CodeAgent pattern: the model emits code that runs in a sandbox (e.g., E2B). Only selected return values flow back to the LLM, keeping large objects — images, audio, binary data — isolated in the environment.

State Schema Isolation

An agent's runtime state object can contain multiple fields. The developer decides which fields are exposed to the LLM at each step, isolating tool-call outputs until they are actually needed.

Conclusion

Context engineering is becoming a core craft for agent builders. The four strategies form a complete toolkit:

  • Write — save context outside the window to help an agent perform a task.
  • Select — pull context into the window to help an agent perform a task.
  • Compress — retain only the tokens required to perform a task.
  • Isolate — split context up to help an agent perform a task.

LangGraph makes it straightforward to implement each strategy. LangSmith provides the evaluation loop to measure impact, identify the best opportunity for context engineering, and iterate quickly.
